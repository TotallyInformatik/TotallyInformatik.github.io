Maze Algorithm

This maze algorithm is very simple and finds a solution to a maze. This program took me about 30min - 1hr to make and it uses only one module: turtle, to visualize the path. The program uses a recursive algorithm to solve the given maze. It has a few limitations though. The program just consists of a script. It is not a .exe file.

Features

This program does one thing: It finds the path from entrance to the exit in python. In the script there is a section where you can change the maze. Once you have changed the maze, you can run the script and watch go. If there is no path, it won't display the path, unlike in the following image:

Instead, the "traveller" won't move at all. The console will display an empty list instead of a list of instructions leading you to the exit aswell.

How this program does its thing

This program uses a recursive algorithm to solve the maze. There is a function that, traverses all the adjacent "squares" (top, bottom, left, right) and returns whether or not the adjacent squares are traversable (wall or not). If the square is traversable, the same function is called on that square.

At the end, once one instance of a function reaches the exit, it will return a value of True to the previous iteration and adds the correct instruction (direction to go) to a list that contains all instructions. This process of returning True and adding to the list of instructions goes on until we have reached our first instance of the function and the algorithm is finished.

But what if there is not solution? In that case, the first instance of this function should return False, as all instances that it has called should return False aswell. That way, we know that this maze is impossible to escape and the value False and an empty list is displayed in the console.

In order to prevent the algorithm from traversing a square that has already been traversed and returning True, the script has a list that contains all traversed squares.

All in all, it is a very simple script and still does its job. Sadly, there are a few limitations. If the exit "square" has two adjacent squares that are traversable, the program breaks for some reason. It is probably because two instances of the function start to return True and add their instrctions to the list of instructions. So for now, be sure to not put the exit square in a place like that if you want it to work properly. I might fix it soon.

Conclusion

This script was created much more recently. (as of 09.05.2020) This time I used a class and I immediately realized how much more structured the program becomes once you use classes. I could have made the structure better by adding an "if __name__ == "main": " statement but for now I will leave it.

I was surprised I took this little time to figure out a solution to this problem. It really showed me the power of recursive functions. I also took a long time to actually get started on this script because I kept pushing it away, almost as if I was scared of it. It taught me to go for something if you want to get it done and not to be scared of it just because the problem seems very complex at first.
