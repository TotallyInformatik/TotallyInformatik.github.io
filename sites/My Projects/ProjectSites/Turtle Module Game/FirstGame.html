Dungeon Raiders

This was the first program that I made. The script consists of a few modules (turtle, os, time, random) and 981 lines of code. It was my first attempt at making a video game, and it wasn't really a full game... This game was created some time ago so there are also some immature jokes..

Initially, when I was just getting started with my programming journey, I wanted to make video games. Instead of using a programming language more fitting like C# and Unity, I decided to use Python as it was much easier. This game was heavily inspired by Undertale and one aspect is from Overwatch. The reason I used the turtle module instead of pygame was because I couldn't install any libraries via pip. (I didn't include pip in my path...)

The process of making this program was very very tiring. The main reason was because I had to draw out almost all parts of the UI myself.

Features

The game wasn't fully finished when I decided to move on to other projects and C# and Unity. As of now, there is only a tutorial that teaches you the basic concepts of this game.

The initial idea was:

  • There is a "grid" in which you could move
  • You can move around with wasd
  • There are enemies you can approach and fight
  • fighting works like this:
    • Attack: three targets appear which you will have to click on to deal damage (time limit is variable)
    • Heal: use items from your inventory to heal yourself of boost damage
    • Ultimate: use a heavy attack that deals more damage
    • Defend: dodge incoming objects to avoid damage

When you start the game, you are greeted with a blank menu that I put together towards the end of the project when I lost motivation. One would then use w and s and the enter key to choose between the tutorial and the main game. Sadly, I never made the actual game.

In the tutorial, a text towards the top will walk you through the game features. You could click to advance and you had to fight against a "goblin".

How this program does its thing

I made this program a while ago. The code was way too messy and deciphering it would take some time. Maybe one day I will go into detail about how everything works in this program, but for now, I can only explain things briefly.

The GUI ist made out of turtles with images and lots of things drawn by a pen turtle and the movement is handled by built-in functions of the turtle module. This program is made of a lot of functions which handle different parts of the game: attacking, etc. The tutorial is made with a variable that tracks at which stage you are in the tutorial. By clicking, you update the variable and a new text is shown.

Conclusion

All in all, this program is what a programmer who has just started would typically write. The code was very messy and whitespace was very inconsistent and all over the place.

In hindsight, I should have used more functions and classes to make my code more reusable. That way, making this little game would have taken much less time and my code would have been much more efficient.

After this project, I moved on to other ways of making games, such as coding with Unity, which made a lot of processes in making a game more easy and simple.

Sadly, I think this is my only game that I have made myself. After moving on to Unity, I slowly lost interest in gamedevelopment. The reason why is still not really clear. I bought a Udemy course to learn the basics of 2D and 3D gamedevelopment but for some reason I just didn't wan't to continue after a while.

I think one of the reasons was that there was too little programming and too much messing around in the Unity game engine and gamedesign. I still rememeber thinking to myself that this process of learning was way to stressful because all I was doing was watching tutorials and not actually making a game I wanted to make.

Thinking about it now, I shouldn't have binged that many videos without making my own game. What I should have done is create a small little game myself so that I didn't lose interest in gamedevelopment. As of now (06.05.2020) I will move on to other parts of programming. Maybe next year or so I will come back to this aspect of programming and give it another shot.
